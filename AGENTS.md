# AGENTS.md
## Команди (Commands)
- Инсталация: `pnpm install`
- Локален сървър: `pnpm run dev`
- Линтинг: `pnpm run lint`
- Билд: `pnpm run build`

## Дефиниция за "Готово" (Definition of done)
- След всяка значима стъпка, изпълни lint и typecheck (ако е приложимо).
- Актуализирай документацията в `docs/agent/` винаги, когато проектът се промени съответно.
- Поддържай дневника на промените (`04_changelog.md`) актуален с потвърдени промени.

## Проектни споразумения (Project Agreements)
- Архитектурата и основните решения трябва да следват `docs/agent/01_architecture.md`.
- Преди добавяне на нови зависимости, консултирай се с текущия план и състояние.
- Предпочитай разширяването на съществуващи модули пред ненужно създаване на нови абстракции.
- Основната структура на документацията за агенти е следната:
  - `docs/agent/00_current_state.md`: Текущо състояние и стек.
  - `docs/agent/01_architecture.md`: Архитектура на данните и приложението.
  - `docs/agent/02_commands_and_env.md`: Среда и команди.
  - `docs/agent/03_active_plan.md`: Задачи в процес.
  - `docs/agent/04_changelog.md`: Дневник на промените.
  - `docs/agent/05_module_map.md`: Карта на модулите.
  - `docs/agent/06_known_issues.md`: Известни проблеми и технически дълг.
  - `docs/agent/07_handoff.md`: Документ за предаване на работата.
  - `docs/agent/decisions/`: За бъдещи архитектурни решения (ADR).
  - `docs/agent/archive/`: За стари документи.
